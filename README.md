# 游戏客户端性能监控工具

> 基于 Qt 5 与 C++17 开发的桌面性能分析工具，用于模拟、采集、解析并可视化游戏客户端运行指标。

[下载 Windows 版本与完整源码](https://github.com/7x8bx8xhqq-maker/game-performance-monitor-release/releases/tag/v1.0.0)

## 项目简介

游戏客户端在运行过程中会持续产生帧率、网络延迟、输入响应和单帧耗时等高频数据。本项目模拟这一真实场景，将指标采集、二进制编码、分块传输、协议解析和界面展示组织为一条完整的数据处理链路。

工具既可以作为 Qt 多线程与通信协议课程项目，也可以用于展示游戏客户端性能监控、测试开发和工具链开发方面的工程能力。

## 主要功能

- **实时指标监控**：动态展示 FPS、Ping、输入延迟和帧耗时。
- **趋势曲线绘制**：使用 `QPainter` 自绘多通道曲线，不依赖 Qt Charts。
- **运行状态统计**：展示当前值、峰值、平均值、样本数量和缓冲区深度。
- **自定义二进制协议**：采用帧头、长度字段、载荷和 CRC16 校验组织数据。
- **异常数据处理**：支持粘包、分包、校验失败和异常字节后的重新同步。
- **异步数据流水线**：采集、解析和界面刷新分离，高频处理不会阻塞主界面。
- **线程状态可观测**：界面可查看 UI、Worker、采集和解析线程状态及编号。
- **可调模拟参数**：可控制采样频率、异常数据比例和性能波动强度。

## 技术架构

```text
游戏运行数据模拟
      |
      v
采集线程 -> 二进制编码 -> 模拟网络分块
      |
      v
线程安全有界缓冲区
      |
      v
解析线程 -> 协议校验与重同步 -> Qt Signal
      |
      v
UI 主线程 -> 指标面板 / 实时曲线 / 运行日志
```

采集侧和解析侧通过 `std::deque`、`std::mutex` 与 `std::condition_variable` 协调。缓冲区为空时解析线程休眠，缓冲区达到上限时写入侧等待，从而限制内存增长并减少无效轮询。

## 自定义二进制协议

```text
0xAA55 | Payload Length | Payload | CRC16
```

- `0xAA55`：固定同步头，用于定位一帧数据的起点。
- `Payload Length`：描述载荷长度，用于判断完整数据是否到达。
- `Payload`：保存 FPS、Ping、输入延迟和帧耗时等指标。
- `CRC16`：检查传输过程中是否出现数据损坏。

解析器会保留未完成的数据片段，并在后续字节到达后继续解析；发现错误帧时会扫描下一个同步头，避免单次异常破坏后续数据。

## 游戏数据如何模拟

后台采集线程按照设定频率生成性能样本，并在基础值上加入随机波动、周期变化和偶发尖峰，用于模拟正常运行、网络抖动和瞬时卡顿。随后样本被编码成二进制帧，再随机切分为多个字节块写入缓冲区，从而复现真实网络或进程间通信中的分包情况。

接入真实游戏时，可以将模拟入口替换为客户端埋点、日志回放、共享内存或 `QTcpSocket` 数据，协议解析和可视化层无需大规模修改。

## 技术栈

| 模块 | 技术 |
| --- | --- |
| 桌面界面 | Qt Widgets / Qt Designer |
| 编程语言 | C++17 |
| 并发处理 | `std::thread` / `std::mutex` / `std::condition_variable` |
| 数据协议 | 自定义二进制帧 / CRC16 |
| 实时绘图 | `QPainter` |
| 构建方式 | qmake / CMake |

## 核心模块

- `TelemetryWorker`：负责样本生成、数据分块、缓冲协调和后台解析。
- `Protocol`：负责二进制帧编码、CRC16 校验、粘包分包处理与重同步。
- `ChartWidget`：负责性能曲线的实时绘制。
- `MainWindow`：负责界面交互、统计信息、日志和线程状态展示。
- `MainWindow.ui`：Qt Designer 主界面文件。

## 下载与运行

1. 打开 [v1.0.0 发布页面](https://github.com/7x8bx8xhqq-maker/game-performance-monitor-release/releases/tag/v1.0.0)。
2. 下载 `Project1_GamePerfMonitor_v1.0_Windows.zip`。
3. 将压缩包完整解压到同一目录。
4. 运行 `QtTelemetryWorkbench.exe`。

支持 Windows 10 / 11。请保留 EXE、DLL 和 `platforms` 目录的相对位置。

## 源码构建

发布页中的 `Project1_Source_v1.0.zip` 包含完整 Qt/C++ 源码、`.ui` 文件、工程文件和应用图标。推荐使用 Qt 5.14.2 与 MinGW 7.3 32-bit 工具链。

```powershell
qmake QtTelemetryWorkbench.pro
mingw32-make -j4
```

也可以使用项目内的 `CMakeLists.txt` 进行构建。
